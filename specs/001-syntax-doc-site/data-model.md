# Data Model: Syntax Documentation

**Feature Branch**: `001-syntax-doc-site`

## Overview

The application relies on a static JSON index generated at build time. There is no runtime database.

## Entities

### 1. Document

Represents a single PDF file found in the `/data` directory.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier (hash of filepath or slug). |
| `filename` | `string` | Name of the file (e.g., "00_Intro.pdf"). |
| `path` | `string` | Relative path from data root (e.g., "PART 1/00_Intro.pdf"). |
| `category` | `string` | The parent folder name (e.g., "PART 1 OVERVIEW"). |
| `pageCount` | `number` | Total pages in the document. |
| `lastModified` | `string` | ISO date string of file modification. |

### 2. SearchIndexItem

The unit of granularity for the search index. We index by **page** or **section** to provide precise search results.

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique ID for the index item. |
| `documentId` | `string` | Reference to the parent Document. |
| `pageNumber` | `number` | The page number where this text appears. |
| `content` | `string` | The extracted text content for this page (cleaned/normalized). |
| `title` | `string` | Title of the document (denormalized for search performance). |
| `category` | `string` | Category (denormalized for filtering). |

## File Structure (Generated)

### `public/search-index.json`

This file is generated by the build script and fetched by the client.

```json
[
  {
    "id": "doc_1_page_5",
    "documentId": "doc_1",
    "title": "CPU Module Instructions",
    "category": "PART 3",
    "pageNumber": 5,
    "content": "LD instruction is used for..."
  },
  ...
]
```

## State Management (Client)

- **Search Query**: String input by user.
- **Filtered Results**: Array of `SearchIndexItem` matches.
- **Selected Document**: The currently viewed PDF.
